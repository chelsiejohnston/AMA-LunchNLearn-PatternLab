Successes & Challenges

Successes

  • Concise code
  • Reveal gaps in UX/requirements from VSA - allows us to ID things that we need to simplify
  • Retrospectives, sprint planning, keeping up good rhythm
  • Pair code review is helpful

Challenges

  • Naming things is hard
  • Typography hellscape
  • Defining atoms
  • Setting the precedent is scary
  • Iterating on the precedent - choosing to table decisions with the knowledge
  • Diescerning requirements from implmenetation realities (is this bad because VSA gave us weird requirements, the requirements changed, vendors made it weird, we made it weird, etc.?)
